## 第一章：Markdown写作最佳实践——让内容“长”出结构

### 1.1 用标题构建清晰的信息层级

Markdown语法中，`#` 号的数量表示标题级别。我的一条重要经验是：**一篇文档只用一级标题（H1）作为主标题，且只出现一次**。H2作为章节标题，H3作为子章节，H4及以下尽量少用。为什么？因为搜索引擎（和读者）会通过标题层级快速理解文章脉络。

```
# 文章主标题（唯一H1）
## 第一章：基础语法（H2）
### 1.1 标题层级（H3）
### 1.2 列表用法（H3）
## 第二章：进阶技巧（H2）
```

### 1.2 善用列表、代码块与引用

- **无序列表**：用 `-` 或 `*` 表示，适合并列的信息点。
- **有序列表**：用 `1.` `2.` 开头，适合步骤或优先级场景。
- **代码块**：用三个反引号 ``` 包裹，并标注语言（如 ```` ```python ````）。注意代码块内不要用短横线代替反引号——这个细节很多人搞混。
- **引用**：用 `>` 开头，适合引用他人观点或重要说明。

**我个人最常用的实战技巧**：每次写技术教程时，都用代码块展示“Markdown语法示例”和“渲染效果”对比。这样读者既能看懂语法，又能预见输出结果。

### 1.3 图片与链接的“有备无患”

在Markdown中插入图片和内链都很简单，但最佳实践是**提前规划图片的存放位置**。如果使用本地图片，迁移或分享文档时容易丢失。所以我强烈建议：**使用支持图床的Markdown编辑器**，或者确保编辑器能自动上传图片到云端。这也是为什么我后面会特别推荐在线Markdown编辑器——它们通常集成了图片托管功能。

### 1.4 永远保持空行与缩进的一致

Markdown对空行和缩进很敏感。比如：

- 列表项之间如果没有空行，会被视为同一个列表的子项。
- 代码块和普通文本间一定要有空行，否则可能渲染异常。

一个简单习惯：**写完一个段落或一个块元素后，都空一行**。这会让你的Markdown源码清晰得像写代码一样。

### 3.2 常见错误排查：Markdown语法常见陷阱

- **标题后要不要加空格**？必须加。`# 标题` 正确，`#标题` 可能不被识别。
- **列表嵌套怎么缩进**？用2个或4个空格（取决于编辑器）缩进下一级。在线Markdown编辑器通常能自动识别。
- **图片加载失败怎么办**？检查图片链接是否是公网可访问的URL，不能用本地路径。如果使用图床，确保图片上传成功。
- **代码块内的Markdown符号被渲染**？记住：代码块内所有符号都是原样显示，不会被解释为Markdown语法。

### 3.3 进阶技巧：为Markdown写一份“样式指南”

如果你经常写文章，我建议你建立一个自己的“Markdown样式指南”文件。例如：

- 标题：最多三级，H1唯一，H2段落开头，H3用于子项。
- 列表：无序列表前加空行，有序列表与无序列表不混用。
- 代码块：一律使用带语言标注的围栏代码块（``` ```）。
- 代码行内：用单个反引号包起来。
- 引用：只用一层，不嵌套。
